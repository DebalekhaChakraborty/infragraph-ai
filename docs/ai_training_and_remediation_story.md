# AI Training And Remediation Story

InfraGraph AI converts static infrastructure diagrams into graph memory, simulates topology-aware incidents, ranks root causes over connected diagrams, and generates safe remediation plans grounded in graph evidence.

## Architecture Story

1. Diagram Intelligence extracts devices, labels, IPs, connectors, and local graph edges from topology diagrams.
2. Topology RCA performs single-diagram graph reasoning over one local graph.
3. Enterprise Graph Brain stitches related local graphs into enterprise graph memory.
4. Enterprise GNN RCA performs cross-diagram graph reasoning when a trained result file exists for the selected scenario.
5. Vector memory indexes graph evidence, incident timelines, RCA outputs, GNN rankings, and AI resolution plans in ChromaDB.
6. Qwen/vLLM generates remediation plans from RCA context plus retrieved graph evidence.
7. The `training/verl_grpo` scaffold prepares sample LoRA + GRPO/vERL alignment records for Qwen3 on AMD GPUs.

## Qwen Remediation

The remediation agent receives root cause, impacted nodes, impacted diagrams, impact path, alert timeline, RCA source, GNN ranking when available, and retrieved evidence IDs. The prompt requires JSON-only output with validation before remediation, rollback/safety notes, blast-radius reasoning, and no invented devices or services.

The app labels response source clearly:

- `Live Qwen/vLLM`: generated by the local model server.
- `Template fallback — deterministic, not model-generated`: generated by rules when Qwen is unavailable.
- `Strict mode blocked`: no model response and deterministic response was not approved.

## Vector Memory

ChromaDB stores semantic chunks with evidence IDs such as `E001`, including device memory, connector memory, interface/IP evidence, OCR text, incident alerts, RCA results, GNN ranking, AI resolution sections, and scenario summaries.

Graph Copilot retrieves evidence cards before answering. When retrieved evidence is present, Qwen/Copilot prompts ask the model to cite evidence IDs.

## GNN To Remediation

Enterprise GNN RCA result files are used only when they exist for the exact selected scenario. Otherwise the app labels the flow as `Scenario-grounded RCA simulation` and does not claim GNN inference. The remediation prompt includes the RCA source so Qwen can explain whether ranking came from GNN or scenario evidence.

## LoRA + GRPO/vERL Package

`training/verl_grpo` is an alignment pipeline scaffold:

- `build_rca_rl_dataset.py` writes train/eval JSONL records.
- `reward_functions.py` scores chosen and rejected responses.
- `sample_config.yaml` documents small-batch Qwen3 LoRA + GRPO settings.
- `train_qwen3_grpo.sh` checks for vERL and prints the training command when vERL is unavailable.

This is not a completed full-scale training run. It is a runnable preparation layer showing how InfraGraph evidence becomes alignment data.
